# MemoryLane Frontend Deployment Guide (Vercel)

This document provides instructions for deploying the Next.js frontend to Vercel.

## Prerequisites
1. A Vercel account.
2. A deployed and running MemoryLane backend API.

## Step 1: Create a Vercel Project
1. Log in to [Vercel](https://vercel.com).
2. Click **Add New** > **Project** > Import `memorylane-frontend`.
3. Set the Framework Preset to **Next.js**.

## Step 2: Configure Environment Variables
In the **Environment Variables** configuration, add the following variables:

```env
# Supabase
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY=your-anon-public-key

# Razorpay
NEXT_PUBLIC_RAZORPAY_KEY_ID=your_razorpay_key_id

# Backend URL (The exposed public domain generated by Railway)
NEXT_PUBLIC_API_URL=https://your-backend.up.railway.app
```

## Step 3: Deploy
1. Click **Deploy**. Vercel will build the production bundle and assign a public domain.
2. Once the deploy completes, copy the generated public Vercel domain and update the `FRONTEND_URL` in the Railway environment variables so CORS is fully restricted.
